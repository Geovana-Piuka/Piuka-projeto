# Módulo Frila — Piuka

## O que cobre

Demandas de artes para a designer freelancer (Isabelle): peças de loja física, materiais impressos, banners, PVCs, layouts. Output vai para Isabelle via Marcela.

Seções de origem no instalador: seção 2.5 (briefing visual Isabelle), seção Marcela (2.6 — loja física).

**Diferença entre Frila e Imagens:**
- Módulo Imagens → artes digitais geradas via IA (/imagens-piuka)
- Módulo Frila → artes para impressão/loja física feitas pela Isabelle (designer humana)

---

## Perguntas de extração

### Bloco 1 — Volume e prioridade

```
→ Quais artes a Isabelle precisa fazer este mês?
→ Qual a mais urgente — o que trava se não sair primeiro?
→ Tem arte que depende de foto ainda não produzida? (sinalizar dependência)
```

### Bloco 2 — Ficha por arte

Para cada arte, coletar:

```
→ Nome da arte (ex: "Banner Vitrine Namorados", "PVC Coleção Inverno", "Layout Home Site")
→ Destino: loja física / site / WhatsApp / e-mail / anúncio impresso / outro
→ Formato/dimensões: [medida em cm ou px — confirmar com Isabelle se não souber]
→ Quantidade: [ex: 2 unidades, 1 por loja, 3 tamanhos]
→ Data de entrega limite
→ Produto em destaque: [nome + foto disponível: SIM / NÃO — produzir]
→ Headline obrigatória: [texto exato ou "Isabelle define"]
→ Copy de apoio: [texto ou "sem copy"]
→ CTA: [texto ou "sem CTA"]
→ Restrição visual: [o que não pode aparecer]
→ Referência: [link de campanha anterior / descrição / ⚠️ pendente]
```

## Situações comuns

**Dimensões não confirmadas:**
"⚠️ Dimensões: confirmar com Isabelle antes de iniciar. Não avançar na arte sem medida fechada."

**Foto não disponível:**
Sinalizar dependência no output: "⚠️ Esta arte depende das fotos da sessão de [data] — só pode começar após aprovação das fotos."

**Prazo apertado:**
"Com o prazo curto, qual arte precisa sair primeiro? Vou ordenar o briefing por prioridade."

**Arte de loja física (sempre incluir no output para Marcela):**
```
TAREFAS MARCELA — LOJA FÍSICA:
→ Confirmar que material impresso chegou antes de [data]
→ Verificar se preço do PDV confere com o preço do site
→ Informar equipe da loja sobre produto foco e oferta ativa
→ Confirmar disponibilidade do produto na loja antes de publicar
```
